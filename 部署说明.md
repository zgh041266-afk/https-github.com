## 🚀 部署步骤（3步）

### 第1步：上传代码到 GitHub

```bash
# 在项目根目录执行
cd legal-doc-demo

# 初始化git（如果还没初始化）
git init
git add .
git commit -m "Initial commit"

# 在GitHub创建新仓库，然后关联推送
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

> 💡 **或者直接用 GitHub Desktop** 拖拽上传更简单

---

### 第2步：在 Vercel 导入项目

1. 打开 https://vercel.com
2. 点击 **"Add New Project"**
3. 选择刚才推送的 GitHub 仓库
4. 点击 **"Import"**

## 🆘 常见问题

### Q: 部署失败怎么办？
A: 检查 Vercel 的 Build Logs，通常是环境变量没配置对

### Q: 如何自定义域名？
A: 在 Vercel Settings → Domains 中添加自己的域名

### Q: 代码保密怎么办？
A: GitHub 仓库设为 Private，Vercel 也支持私有仓库
